# Introducción al modelado de regresión y al marco PACE

## Contexto general

Es posible que ya esté familiarizado con términos como **modelos de aprendizaje automático** o **análisis de regresión**. Si no es así, este programa está diseñado para guiarle desde los fundamentos. El objetivo es iniciar un viaje de aprendizaje enfocado en comprender conceptos clave y su aplicación práctica.

## Modelos de regresión

- Se basan en una **fundamentación estadística**.
- Utilizan datos existentes para **inferir o predecir** el comportamiento de otros datos.
- Su propósito principal es **contar una historia** sobre las relaciones entre variables.
- Estas historias ayudan a las **partes interesadas** a tomar mejores decisiones estratégicas y empresariales.
- El modelado es un **proceso iterativo**, similar a otros marcos como el ciclo de vida de los datos o el análisis exploratorio de datos (EDA).

## El marco PACE

En este curso se utiliza el marco **PACE**, que significa:
- **Planificar (Plan)**
- **Analizar (Analyze)**
- **Construir (Construct)**
- **Ejecutar (Execute)**

PACE proporciona una estructura clara para realizar análisis de regresión y guía todo el proceso de modelado.

## Etapas de PACE en el análisis de regresión

### 1. Planificar

- Consiste en **comprender los datos en el contexto del problema**.
- El conocimiento del dominio (industria, negocio u otros contextos) es clave.
- Se analizan aspectos como:
  - Qué datos están disponibles.
  - Cómo se recopilaron.
  - Cuáles son las necesidades empresariales.
- Esta etapa **guía a las demás fases** del proceso.

### 2. Analizar

- Se exploran los datos en profundidad para **seleccionar modelos apropiados**.
- En regresión, se utiliza **Python** para:
  - Realizar análisis exploratorio de datos (EDA).
  - Verificar los **supuestos del modelo**.
- Los supuestos del modelo son condiciones que deben cumplirse para aplicar ciertas técnicas estadísticas.
- Una sólida base en **estadística** permite construir modelos significativos y confiables.

### 3. Construir

- Se desarrolla el modelo utilizando Python u otro lenguaje de programación.
- Incluye:
  - Selección de variables.
  - Transformación de datos.
  - Escritura del código.
- Los supuestos del modelo se **revisan nuevamente** después de construirlo.
- Se evalúan los resultados respondiendo preguntas como:
  - ¿Qué tan bueno es el modelo?
- Se eligen métricas, se comparan modelos y se obtienen resultados preliminares.
- El EDA puede repetirse para **refinar el modelo**.

### 4. Ejecutar

- Se interpretan los resultados para **contar la historia final**.
- Se preparan:
  - Visualizaciones.
  - Resultados formales.
- Las estadísticas del modelo se traducen en **afirmaciones comprensibles** sobre la relación entre variables.
- Las conclusiones deben alinearse con el contexto y las preguntas planteadas en la fase de planificación.

## Principios clave del marco PACE

- Los **datos están en el centro** de todo el proceso.
- PACE ayuda a los profesionales de datos a mantenerse **organizados y rigurosos**.
- Los conocimientos generados deben:
  - Basarse en datos.
  - Ser precisos.
  - Tener sentido dentro del contexto empresarial o comunitario.
- PACE es **iterativo**:
  - Las etapas pueden repetirse o cambiar de orden según la situación.
  - La experiencia profesional guía cuándo y cómo moverse entre fases.
